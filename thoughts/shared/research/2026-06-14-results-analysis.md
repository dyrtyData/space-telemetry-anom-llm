## 1. What the project was actually asked to prove

The brief came from a friend evaluating whether the author could deliver a **mission-critical**
capability for his product: **(a) detect anomalies** in a telemetry-like multivariate stream, and
**(b) give an end user human-readable advice on the effect of a change** — without depending on an
external API vendor. The stated hiring signal was *"the ability to move beyond external API
wrappers and successfully fine-tune open-source foundation models for highly specific, localized
business use cases."*

Two architectural philosophies were on the table in the exploratory thread:

1. **Two-stage (industry standard):** a cheap traditional detector triggers; an LLM explains.
2. **Unified LLM (research frontier):** a single fine-tuned LLM tokenizes the numeric window and
   both detects *and* advises (the AnomLLM / Time-LLM line of work).

The decision taken in the plan was **to build and measure both**, plus a classical
non-deep-learning floor (Isolation Forest), so the comparison is empirical rather than asserted.
That is the spine of how these results should be read: this is a **bake-off**, and the headline
result is a *finding*, not a *win for the shiny option*.

---

## 2. The headline result

Final numbers on the ESA-AD test material (test split = 4,500 windows, 25.0% anomalous):

| Approach | Precision | Recall | F1 | CEF0.5 | Affinity-F1 | Eval unit |
|---|---|---|---|---|---|---|
| Isolation Forest | 0.127 | 0.459 | 0.188 | 0.149 | N/A | 3 channels (macro) |
| **LSTM baseline** | **0.835** | 0.552 | **0.663** | **0.757** | N/A | 3 channels (macro) |
| LLM detection (Qwen3-8B, QLoRA→GGUF) | 0.360 | **0.609** | 0.453 | 0.392 | 0.456 | 4,500 windows (micro) |
| Hybrid (LSTM detect + LLM advice) | 0.835 | 0.552 | 0.663 | 0.757 | N/A | 3 channels (macro) |

**The fine-tuned LLM did not beat the LSTM at detection.** The LSTM is roughly **2.3× more
precise** (0.835 vs 0.360) and wins F1 (0.663 vs 0.453) and the precision-weighted CEF0.5
(0.757 vs 0.392) decisively. The LLM's only detection edge is **recall** (0.609 vs 0.552) — it
catches more true anomalies, but at the cost of a flood of false alarms.

This is the honest, senior-level outcome, and it *matches the literature*: direct LLM time-series
detection trades precision for recall and is not yet competitive with a tuned sequence model on a
clean, channel-specific benchmark. **The contribution of this project is demonstrating that
trade-off on real ESA data, with reproducible numbers — and drawing the right architectural
conclusion from it.**

## 3. Reading the LLM confusion matrix

n=4,500 windows: **TP=684, FP=1,214, FN=439, TN=2,161**, accuracy 0.632, 27 unparsed (`UNKNOWN`).

- The model is **over-eager**: it raises 1,898 anomaly flags (684 + 1,214) against 1,123 true
  anomalies. False positives (1,214) outnumber false negatives (439) ~2.8:1.
- At a 25% base rate, **precision of 0.36 is still well above the 0.25 random-flag floor**, and
  recall of 0.61 is genuine signal — the model has learned *something* about the rhythm of
  nominal vs. anomalous telemetry. It is not guessing.
- But 0.36 precision means **~2 of every 3 alarms are wrong**. In a mission-critical control room
  that is an alert-fatigue generator. As a *trigger*, this model is not deployable. As a
  *recall-maximizing second opinion* layered behind a high-precision detector, it could be useful.
- Only 27/4,500 (0.6%) responses failed to parse to ANOMALY/NOMINAL — the fine-tune reliably
  produces the structured output format it was trained on.

## 4. The crucial caveat: this is not yet apples-to-apples

The single most important thing an interviewer should know (and the report says so openly) is
that **the LSTM and the LLM were measured on different evaluation units**:

- **LSTM / Isolation Forest:** scored per channel and **macro-averaged over just 3 Mission-1
  target channels** (`channel_12/13/14`), each with its **own tuned anomaly threshold**. This was
  a Phase-2 *smoke* run, never expanded to all 58 Mission-1 target channels (let alone all 224).
- **LLM:** scored per window, **micro-averaged over 4,500 windows spanning all three missions and
  all channels**, with **no per-channel tuning** — one global model, one decision rule.

So the LSTM enjoyed three advantages the LLM did not: (1) an easier, hand-picked channel subset,
(2) per-channel threshold calibration, and (3) macro-averaging that hides hard channels. The LLM
faced the full, heterogeneous, untuned distribution. **The gap is real, but it is an upper bound
on the LSTM's true advantage, not a like-for-like measurement.** A fair rematch would either:
- expand the LSTM to all 58 Mission-1 target channels (`make baseline … --max-channels 58`), and/or
- evaluate both on an identical, contiguous (un-shuffled) per-channel timeline.

This is flagged as the top follow-up in both the plan and the log. It is called out here rather
than buried because *claiming the LSTM "won" without this caveat would be the kind of sloppy
benchmarking the brief explicitly warns against.*

A second methodological honesty note: **Affinity-F1 is degenerate on this test split.** The split
is shuffled and balanced-subsampled (~1.4 windows per channel), so interval reconstruction yields
mostly isolated single-window "intervals" and Affinity-F1 (0.456) collapses to ≈ window-level F1.
The metric is implemented correctly and becomes meaningful only on a contiguous evaluation
stream; it is reported with that disclaimer rather than dressed up as an independent result.

## 5. Where the LLM actually earns its place: the advice layer

The detection bake-off is only half the brief. The other half — *"end user advice on the effect
of a change"* — is something **none of the traditional baselines can do at all.**

- **99.6%** of the LLM's anomaly predictions emitted **structured `DIAGNOSIS` + `ADVICE`** text
  (the trailing `ACTION:` line is present too but is clipped by the 300-char persistence cap, so
  coherence was keyed on the two fields that survive truncation).
- This is the capability the fine-tune was built for, and it works reliably. The QLoRA SFT taught
  Qwen3-8B the *format, tone, and domain vocabulary* of spacecraft fault advice from the 7,457
  in-house advice records generated in Phase 1.5.

This reframes the whole result. The LLM is **not** competing with the LSTM to be the trigger. Its
job is to turn a raw flag into an actionable, human-readable diagnosis — and at that it succeeds.

## 6. The architecture the data actually recommends: the Hybrid

Putting §2–§5 together, the empirically-supported recommendation is exactly the **two-stage
hybrid** the industry uses and the exploratory thread anticipated:

```
telemetry window ──► LSTM detector (P=0.835, ~instant, cheap)
                         │ flags an anomaly
                         ▼
                     Qwen3-8B advisor (structured DIAGNOSIS / ADVICE / ACTION)
                         │
                         ▼
                     operator-ready alert
```

- **Detection** is owned by the high-precision, sub-millisecond LSTM — so you don't run an 8B
  model on every window (the LLM costs **2.77 s/window** on M3 Max Metal; at telemetry scale that
  is a non-starter for first-pass screening, an important systems-level point).
- **Explanation** is owned by the LLM, invoked only on the (few) windows the detector flags — so
  the expensive model runs ~1/N of the time and adds the one capability the detector lacks.
- The Hybrid row inherits the LSTM's detection metrics *by construction* (0.835 P / 0.663 F1) and
  layers on 99.6%-structured advice. **It strictly dominates either component alone for the stated
  business need.**

That is the deliverable that answers the friend's actual question: a localized, open-source,
no-external-API system that both detects and advises, with the cost profile to run in production.

correct finding is the point.

## 8. Honest limitations (state these before an interviewer does)

1. **No "did the fine-tuning help?" comparison — the biggest gap.** The fine-tuned LLM was never
   run head-to-head against the **un-fine-tuned base Qwen3-8B**, nor against an off-the-shelf
   model (Claude/GPT), on the same 4,500 windows. For a project whose headline skill is *"I can
   fine-tune,"* the single cleanest proof — base-model vs. fine-tuned-model on identical test data
   — is missing. We can show the *pipeline* works; we have not yet isolated the *delta* the
   fine-tune contributed. **This is the top recommended follow-up.**
2. **The Hybrid is a construction, not an independent measurement.** Its detection P/R/F1 equal
   the LSTM's by definition (see §6); the only thing genuinely new in the Hybrid is the advice
   layer, which is graded only structurally (#5 below). It was not benchmarked against any
   alternative advice generator.
3. **Scope cut — the vision detector was never run.** The original research design
   (`2026-06-12-star-pipeline-codebase-research.md`) specified "LLM Detection" as a **vision**
   model: Qwen3-VL-8B fine-tuned on **PNG plots** of telemetry windows ("AnomSeer-style").
   `train_detection.py` was written but **not trained or run** — this run was advice-(text)-model
   only (implementation log D-notes + Phase-3 scope). The approach evaluated here as "LLM
   detection" is therefore the *text* advice model asked to also classify ANOMALY/NOMINAL, **not**
   the planned vision approach. A reasonable scope decision, but it means the project shipped 2 of
   the 3 originally-planned LLM approaches.
4. **Baselines are a 3-channel smoke run.** The LSTM's strong numbers are on an easy, tuned
   subset. Until it's run on all 58 target channels, the head-to-head is an upper bound on its
   advantage, not a settled result.
5. **Different eval units** (3 channels macro vs. 4,500 windows micro) — see §4.
6. **Affinity-F1 is degenerate** on the shuffled split — see §4.
7. **Advice quality is structural, not yet semantic.** We verified 99.6% of advice has the right
   *shape* (DIAGNOSIS/ADVICE/ACTION). We did **not** grade whether the advice is *correct* — no
   expert review, no LLM-as-judge factuality scoring, no grounding check against the anomaly's
   true root cause. The advice labels were synthetically generated in-session.
8. **Advice persistence truncates at 300 chars**, clipping the ACTION line in the saved record
   (the model generates it; we just don't store the tail).
9. **Single fine-tune, no hyperparameter sweep.** r=16/α=16/3-epoch was taken from the plan's
   sensible defaults; no ablation over LoRA rank, epoch count, or prompt format was run.
10. **No detection-tuned LLM variant.** The deployed model was the *advice* SFT; a detection-only
    SFT or a calibrated decision threshold on the LLM's output could shift its precision/recall
    operating point and was not explored.

---

## 9. Recommended next steps (in priority order)

1. **Prove the fine-tune helped (highest value):** run the *un-fine-tuned* base Qwen3-8B over the
   same 4,500 windows and report the delta in detection + advice. Optionally add an off-the-shelf
   model (Claude/GPT) as a "could you have just used an API?" reference. This directly answers the
   project's core skill claim and is currently missing.
2. **Level the detection field:** run the LSTM on all 58 Mission-1 target channels and re-score;
   ideally evaluate both LSTM and LLM on an identical contiguous per-channel stream so Affinity-F1
   becomes meaningful.
3. **Grade advice semantically:** add an LLM-as-judge (or expert spot-check) pass scoring advice
   factuality and actionability against the known anomaly root cause — turn "99.6% structured"
   into "X% correct."
4. **Tune the LLM's operating point:** calibrate a decision threshold / prompt to trade some
   recall for precision; report a P-R curve instead of a single point.
5. **Ship the Hybrid as the reference design:** it's the architecture the numbers support and the
   one the business actually needs.
6. **(Optional) Complete the vision approach:** train the written-but-unrun `train_detection.py`
   (Qwen3-VL-8B on PNG plots) to round out the original three-way LLM comparison.
7. **LoRA ablation:** small sweep over rank / epochs / prompt format to confirm the fine-tune is
   near its achievable ceiling.

## 10. One-paragraph takeaway

On real ESA satellite telemetry, a classical LSTM with per-channel error thresholding remains the
stronger *detector* (precision 0.835, F1 0.663) than a QLoRA-fine-tuned Qwen3-8B used as a direct
detector (precision 0.360, F1 0.453) — the LLM buys recall at a steep precision cost and is far
too slow (2.77 s/window) to screen every window anyway. But the LLM delivers the capability the
baselines fundamentally cannot: reliable, structured, human-readable diagnostic advice (99.6% of
flags). The correct, empirically-grounded architecture is therefore the **hybrid** — a cheap
high-precision detector that triggers an LLM advisor — which is exactly the localized,
open-source, no-vendor system the original brief set out to validate. The headline is not "LLMs
win" or "LLMs lose"; it is *"here is the measured trade-off, and here is the design it dictates."*
